# 智能眼镜控制端硬件设计

## 一、硬件目标

智能眼镜控制端的硬件目标是实现一个可穿戴的最小控制端。

第一版不追求外观完整，不做 AR 显示和摄像头识别，只需要能够稳定完成：

* 采集头部姿态
* 判断基础动作
* 显示当前状态
* 通过串口或 MQTT 输出控制命令

## 二、推荐硬件清单

| 硬件    | 推荐型号            |  数量 | 作用             |
| ----- | --------------- | --: | -------------- |
| 主控开发板 | ESP32-S3-N16R8  |   1 | 主控、WiFi、MQTT   |
| 姿态传感器 | MPU6050         |   1 | 加速度和角速度采集      |
| 显示屏   | 0.96 寸 I2C OLED |   1 | 显示模式、手势和命令     |
| 按键模块  | 轻触按键模块          | 2-3 | 模式切换、确认、停止     |
| 蜂鸣器   | 有源蜂鸣器模块         |   1 | 状态提示           |
| 连接线   | 杜邦线             |  若干 | 模块连接           |
| 原型板   | 面包板 / 洞洞板       |   1 | 临时搭建           |
| 固定结构  | 眼镜框 / 头部支架      |   1 | 固定 MPU6050 和主控 |

## 三、为什么主控选择 ESP32-S3

ESP32-S3 适合作为智能眼镜控制端主控，原因是：

* 支持 WiFi，方便后续接入 MQTT
* 性能比普通 STM32F103 更适合处理通信和 JSON
* ESP-IDF 支持 MQTT、OTA 等物联网功能
* 适合作为 AIoT 可穿戴控制端
* 后续可以扩展 OTA、低功耗、更多传感器

## 四、模块连接关系

第一版连接关系：

```text
ESP32-S3
├── I2C → MPU6050
├── I2C → OLED
├── GPIO → 按键模块
├── GPIO → 有源蜂鸣器
└── WiFi / MQTT → Linux IoT Gateway
```

## 五、引脚规划

当前先做文档规划，具体引脚后续根据开发板实际丝印确认。

| 模块      | 接口            | 说明         |
| ------- | ------------- | ---------- |
| MPU6050 | I2C SDA / SCL | 姿态数据采集     |
| OLED    | I2C SDA / SCL | 状态显示       |
| Button1 | GPIO 输入       | 模式切换       |
| Button2 | GPIO 输入       | 停止 / 确认    |
| Buzzer  | GPIO 输出       | 提示音        |
| MQTT    | WiFi          | 上传 JSON 命令 |

## 六、供电说明

第一版可以使用 USB Type-C 给 ESP32-S3 供电。

注意：

* MPU6050 和 OLED 优先使用 3.3V
* 不要把 5V 直接接到只支持 3.3V 的模块信号脚
* 所有模块 GND 必须共地
* 蜂鸣器如果电流较大，后续可以用三极管驱动

## 七、MVP 不做的内容

第一版不做：

* AR 光学显示
* 摄像头视觉识别
* 语音识别
* 蓝牙 Mesh
* 云平台
* 完整外壳
* 超低功耗优化
* 复杂 AI 推理

第一版重点是：

```text
MPU6050 姿态识别
OLED 状态显示
按键模式切换
gesture → cmd
串口 / MQTT 输出
```
